Dataset Distributions and the Caesar Cipher

This assignment explores the concept of statistical distributions through both univariate and bivariate analysis, integrating database operations in Microsoft Access and applying letter frequency analysis for simple cryptographic decoding.

1. Dataset and Distribution Concepts

In statistics, a dataset is a structured collection of data, usually represented in tabular form with variables (columns) and observations (rows). Each variable can exhibit different types of distributions, such as uniform, normal, or skewed, depending on how its values are spread.

A univariate distribution involves a single variable, focusing on characteristics such as its central tendency (mean, median) and dispersion (variance, standard deviation). A bivariate distribution, on the other hand, examines two variables simultaneously, often revealing correlations or dependencies between them.

For example, in an academic dataset of students, one might analyze the distribution of Math Scores as a univariate case and the relationship between Math Scores and Study Hours as a bivariate case.

2. Univariate Distributions in Microsoft Access

To compute measures such as mean and standard deviation directly from a dataset in Microsoft Access, we can use built-in aggregate functions (Avg() and StDev()) in SQL queries. 


SELECT 'Math Score' AS Variable,
       Round(Avg([MathScore]), 1) AS Mean,
       Round(StDev([MathScore]), 1) AS StdDev
FROM Students
UNION ALL
SELECT 'Study Hours', 
       Round(Avg([StudyHours]), 1),
       Round(StDev([StudyHours]), 1)
FROM Students;

This query returns a concise table showing the mean and standard deviation for each variable in the dataset. In our example, Math Score might have an average of 82.4 and a standard deviation of 8.6, while Study Hours might average around 4.6 with a deviation of 1.5.

3. Letter Distribution and the Caesar Cipher

The Caesar cipher is one of the simplest forms of encryption, named after Julius Caesar, who used it to encode military messages. It shifts each letter of the alphabet by a fixed number of positions. For example, with a shift of 3, A → D, B → E, and so on.

Decoding can be achieved either by trying all 25 possible shifts (brute force) or by using letter frequency analysis. Since different languages have characteristic letter distributions (for English, E is most frequent), comparing the ciphertext’s frequency distribution against English norms can reveal the correct shift. 


# Simple Python example: automatic Caesar cipher decoding using frequency analysis
from collections import Counter

cipher = "Lw zdv d uhohqwohvv vdydjh rqvodxjkw ri guxgjhub dqg reolylrq"

def decrypt_caesar(s, k):
    out = []
    for ch in s:
        if ch.isalpha():
            base = 'a' if ch.islower() else 'A'
            out.append(chr((ord(ch) - ord(base) - k) % 26 + ord(base)))
        else:
            out.append(ch)
    return ''.join(out)

# Try all shifts and print those resembling English
for k in range(26):
    print(k, decrypt_caesar(cipher, k))

Running this script prints every possible decryption (shift 0–25). The version that forms coherent English text corresponds to the correct shift—here, a shift of 3, yielding:

“It was a relentless savage onslaught of drudgery and oblivion.”
