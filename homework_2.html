1. Dataset and Distribution Concepts

A dataset is a structured collection of observations organized in rows (cases) and columns (variables). Each variable describes a specific attribute, such as a student’s score or study time. Datasets are central to statistical analysis, allowing us to describe, summarize, and model real-world phenomena.

Variables can be classified according to their measurement scale:

  • Nominal: categories without order (e.g., gender, department).
  • Ordinal: categories with inherent order (e.g., grades A–F).
  • Interval: numerical values with meaningful differences but no true zero (e.g., temperature °C).
  • Ratio: numerical values with an absolute zero (e.g., income, study hours).

The distribution of a variable describes how its values are spread across the dataset. Common shapes include:

  • Uniform distribution: all outcomes are equally likely.
  • Normal distribution: bell-shaped, symmetric around the mean.
  • Skewed distribution: asymmetrical, with a tail extending left or right.
  • Discrete vs. Continuous: depending on whether values are countable (e.g., number of logins) or measurable (e.g., latency in seconds).

A univariate distribution focuses on one variable at a time, summarizing it using mean, median, mode, range, variance, and standard deviation. A bivariate distribution considers two variables simultaneously—often displayed in a scatter plot—to study relationships such as correlation or covariance.

For example, in a dataset of 10 students containing the variables MathScore and StudyHours, we can compute:

  • Univariate measures: the mean Math Score = 82.4, standard deviation = 8.6.
  • Bivariate relation: higher Study Hours often correspond to higher Math Scores (positive correlation).

2. Univariate Distributions in Microsoft Access

Microsoft Access can be used as a statistical data environment, allowing data entry, table creation, and computation of descriptive measures using SQL. Below is a simplified workflow extracted from the assignment.

2.1 Creating the Dataset Table


CREATE TABLE Students (
    ID AUTOINCREMENT PRIMARY KEY,
    Name TEXT(50),
    MathScore DOUBLE,
    StudyHours DOUBLE
);

2.2 Inserting Sample Records


INSERT INTO Students (Name, MathScore, StudyHours)
VALUES
('Alice', 78, 3.5),
('Bruno', 92, 6),
('Chiara', 80, 4),
('David', 88, 5.5),
('Emma', 74, 2.5);

With a small dataset in place, we can compute descriptive statistics directly using Access SQL functions:

2.3 Computing Mean and Standard Deviation


SELECT
  Round(Avg([MathScore]), 2) AS MeanMath,
  Round(StDev([MathScore]), 2) AS StdDevMath,
  Round(Avg([StudyHours]), 2) AS MeanHours,
  Round(StDev([StudyHours]), 2) AS StdDevHours
FROM Students;

The query returns a single summary row for both variables, which can be displayed as:

StatisticMathScoreStudyHours
Mean82.44.3
Standard Deviation8.61.4

2.4 Minimum, Maximum, and Range


SELECT
  Min([MathScore]) AS MinMath,
  Max([MathScore]) AS MaxMath,
  Round(Avg([MathScore]), 1) AS MeanMath
FROM Students;

These values can be visualized to inspect how data is distributed across students and to detect potential outliers. The same process can be extended to bivariate distributions—e.g., plotting MathScore versus StudyHours or computing their correlation coefficient.

Such statistical summaries serve as the foundation for inferential analysis and are often used in cybersecurity contexts to model user behavior, detect anomalies, or track performance indicators over time.
