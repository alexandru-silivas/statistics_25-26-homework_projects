Public Key Cryptography and the RSA Algorithm

This assignment explores the concept of public-key encryption through the RSA algorithm. It covers the mathematical foundations of key generation, encryption, and decryption, demonstrating how two keys (public and private) ensure secure data transmission. The project also includes an interactive RSA tool implemented in JavaScript that allows you to generate keys, encrypt and decrypt messages, and visualize letter distributions—mirroring the statistical approach seen in the Caesar Cipher assignment.

2. How RSA Works — Step by Step

  1. Choose two distinct prime numbers p and q.
  2. Compute n = p × q. This is the modulus used in both keys.
  3. Compute Euler’s totient function: φ(n) = (p - 1)(q - 1).
  4. Select an integer e (1 < e < φ(n)) such that gcd(e, φ(n)) = 1. This is the public exponent.
  5. Find d such that d × e ≡ 1 (mod φ(n)). This is the private exponent.
  6. Public key: (n, e) — Private key: (n, d).
  7. Encryption: c = me mod n. Decryption: m = cd mod n.

3. Example with Small Primes

Suppose we choose p = 61 and q = 53:

  • n = 61 × 53 = 3233
  • φ(n) = 3120
  • Choose e = 17 → gcd(17, 3120) = 1
  • d = 2753 (because 2753 × 17 ≡ 1 mod 3120)

If we encrypt the message "A" (ASCII = 65), we get c = 6517 mod 3233 = 2790. Decrypting: 27902753 mod 3233 = 65 → "A".
