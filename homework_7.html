Homework 7

Server security updates as random walks — simulation & binomial convergence

This assignment models a server receiving weekly security updates for n weeks. Each week m independent attackers attempt to breach the server, each with success probability p. If any attacker succeeds in a week the server is considered breached for that week; otherwise it is considered secure.

Mathematical model

Let the weekly probability the server stays secure be s = (1 − p)m. The probability of a breach in a week is b = 1 − s.

Over n weeks the number of secure weeks K is distributed as K ~ Binomial(n, s). We assign +1 to a secure week, −1 to a breached week. The total score after n weeks is therefore:

S = (+1)·K + (−1)·(n−K) = 2K − n

So possible total scores are {−n, −n+2, …, n−2, n}. The theoretical probability mass for a final score S = 2k − n (i.e. k secure weeks) is:

P(S = 2k − n) = C(n,k) · sk · bn−k

Interpretation as a random walk

The sequence of weekly +1/−1 outcomes is a simple random walk with biased steps: a +1 occurs with probability s and −1 with probability b. The cumulative sum over time is the walk. By simulating many independent trajectories and counting final scores we can empirically estimate the distribution of S and compare it to the Binomial-based theoretical distribution. As n (and the number of simulated trajectories) grow, empirical frequencies should converge to the theoretical probabilities.
