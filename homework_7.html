Homework 7 — Random Walk Security Model

In this assignment, we model a server that receives security updates weekly. Each week, m attackers independently attempt to breach the system, each with probability p. If the server survives a week, it scores +1. If breached, it scores -1. Over n weeks, this creates a cumulative score that behaves like a random walk.

We simulate trajectories and compare final score distributions to the theoretical binomial distribution, showing convergence as n and m increase.

Panel A — One attacker, 50 weeks

With m = 1 attacker and a low breach probability p = 0.05, the server is usually safe. The security score drifts upward.

Panel B — Balanced walk (50 attackers, 50 weeks, forced fairness)

Even though there are many attackers, we artificially force the probability of surviving a week to be s = 0.5. This yields a symmetric distribution around 0, matching the reference site.

Panel C — 50 attackers, 50 weeks, 500 simulations

Now we let the real formula apply: s = (1 - p)^50. With many attackers, survival becomes rare. The score drifts sharply downward, matching realistic multi-attacker threat models.
